# v0.0.41

> Released: 2026-05-07

UI polish, comment-rendering fixes, and a wider-reach for the Ollama adapter — all shipped and validated on `main`.

## New: Ollama (local) adapter visible in UI

The `ollama_local` adapter ([FID-49](https://github.com/fideliosai/fidelios/pull/63)) is now selectable from the **New Agent** and **Agent Settings** dropdowns under "Ollama (local)". Backend support shipped in v0.0.40; this release wires it into the UI.

- Dedicated Host and Cloud-tier inputs plus the standard instructions file path
- Model dropdown auto-populates via `listOllamaModels` (and accepts manual entries like `kimi-k2.6:cloud` when no `OLLAMA_API_KEY` is configured)
- API zod validator now accepts `adapterType: "ollama_local"`
- Other knobs (`keepAlive`, `numCtx`, `think`, `timeoutSec`) reachable via the existing Extra args mechanism

## UI: model selector available for every adapter

[FID-31](https://github.com/fideliosai/fidelios/pull/54) — every adapter now shows a model selector with provider-friendly labels, not just the cloud-first ones. Local adapters expose creatable model fields so you can paste arbitrary model IDs without leaving the form.

## UI: issue detail view stretches to fill width

[FID-46](https://github.com/fideliosai/fidelios/pull/62) — removed the `max-w-2xl` cap on `IssueDetail`, so long descriptions, plans, and comment threads now use the full pane width on wide screens.

## Comments: HTML entity decoding fixed

[FID-43](https://github.com/fideliosai/fidelios/pull/62) — iOS Telegram clients sometimes encode trailing whitespace as numeric HTML entities (`&#x20;`, `&#xA0;`). When markdown was re-pasted into FideliOS, the `&` got backslash-escaped to `\&#x20;`, and both forms surfaced as literal text in the UI, breaking Board comment threads.

- New `decodeWhitespaceEntities` helper applied at telegram-gateway ingestion
- Migration `0049_fill_decode_whitespace_entities_in_comments.sql` backfills existing rows so previously broken comments render cleanly without manual edits
- Render-side decode added to `MarkdownBody.tsx` to cover any non-Telegram source as well
- 10 unit tests cover hex/decimal forms, NBSP, tab/LF/CR, and the backslash-escaped variant

## @-mention wakes: multi-word agent names + PATCH path

[FID-44](https://github.com/fideliosai/fidelios/pull/62) — fixed two latent bugs in the @-mention wake pipeline:

1. **Resolver regex** broke on multi-word agent names. `findMentionedAgents` now uses longest-match resolution and HTML-decodes the comment body before matching.
2. **PATCH `/issues/:id`** (the reassign + comment combo path used by the UI) didn't surface `wakeCommentId`, so reassigned agents woke without a trigger comment. Both POST and PATCH paths now emit the wake event.

12 resolver tests + 6 route tests cover the fix.
