# v0.0.42

> Released: 2026-05-07

Comment-routing reliability, Ollama adapter Phase 2, and a wave of UI fixes. Seven merged PRs on top of v0.0.41.

## Critical: UI @-mentions now wake agents

[FID-44](https://github.com/fideliosai/fidelios/pull/61) ŌĆö comments posted in the FideliOS UI containing `@AgentName` were silently failing to wake the targeted agent. Root cause was two-fold:

- `findMentionedAgents` regex broke on multi-word names ("Magic Leela") and missed HTML-entity-encoded handles
- `PATCH /issues/:id` (combined reassign + comment) did not surface `wakeCommentId`, so reassigned agents woke without the trigger comment

Fixed with longest-match resolution, HTML decoding, deduped fan-out, and `wakeCommentId` propagation through PATCH. 12 resolver tests + 8 route tests cover single/multi/unknown/non-agent/self-mention scenarios. Telegram-gateway ingestion path is unchanged (no double-firing).

## Ollama (local) adapter ŌĆö Phase 2: in-process tool-calling

[FID-16](https://github.com/fideliosai/fidelios/pull/56) layers a native agent harness on top of the Phase 1 chat adapter, ending FideliOS's longstanding pattern of delegating tool execution to upstream CLIs (Pi, Hermes, Claude Code).

- Native tool-calling loop: parses `message.tool_calls`, executes, feeds back as `{role:tool}`
- Built-in tools: `read`, `write`, `bash`, `grep`, `edit` (mirrors the `pi-local` skill-discovery pattern)
- Workspace `cwd` integration via `worktree-config.ts`
- Concurrency cap singleton (Free=1 / PRO=3 / MAX=10) keyed on `ollama_local + :cloud` model
- `getQuotaWindows()` reports the configured tier cap
- 26 tool tests + 14 concurrency tests; multi-turn loop bounded by `maxTurns` (default 20)

Live smoke against `kimi-k2.6:cloud` is the natural follow-up.

## UI: model routing rules editor

[FID-20](https://github.com/fideliosai/fidelios/pull/55) follow-up to FID-14 (per-task model routing). Adds a collapsible **Model Routing Rules** section to `AgentConfigForm`:

- Per-rule `taskKind` + optional `agentRole` selectors
- Model dropdown (`SelectCombobox`, creatable)
- Per-adapter effort dropdowns
- Add / remove / reorder rows
- Persists as `adapterConfig.modelRouting` (validated by existing `modelRoutingSchema`)
- Empty list = no routing (current default behavior preserved)

## UI: issue documents render as markdown by default

[FID-40](https://github.com/fideliosai/fidelios/pull/59) ŌĆö opening an issue document used to mount a `MarkdownEditor` immediately, leaving Board users staring at the placeholder text "Markdown body" instead of the rendered content. New `DocumentBodySlot` renders `MarkdownBody` in read mode and only switches to `MarkdownEditor` on click / focus / keyboard activation. All draft state (autosave, mentions, image upload, submit) is unchanged.

## Hermes (local) adapter: Ollama models in dropdown

[FID-33](https://github.com/fideliosai/fidelios/pull/57) ŌĆö `hermesLocalAdapter.listModels` now delegates to `listOllamaModels`, so the Permissions & Configuration model dropdown actually shows local + cloud Ollama options instead of an empty list. Removed the redundant free-text Model field that FID-31 had left behind in `HermesLocalConfigFields`.

## Telegram: Board-CEO topic now routes directly to CEO

[FID-38](https://github.com/fideliosai/fidelios/pull/58) ŌĆö voice / text messages in the dedicated Board-CEO Telegram topic used to land in `backlog` with no owner. Plugin manifest bumps to `0.2.0`, declares `issues.create` + `agents.invoke` capabilities, and creates issues with `assigneeAgentId = CEO` + immediate heartbeat invoke. Voice path attempts `transcribeAudio` with a graceful placeholder fallback. Capability addition is purely additive.

## Comments: HTML whitespace entities decoded everywhere

[FID-43](https://github.com/fideliosai/fidelios/pull/60) extends the v0.0.41 fix to the server side:

- `decodeWhitespaceEntities` helper added at telegram-gateway ingestion
- Render-side decode in `MarkdownBody.tsx` for non-Telegram sources
- Migration `0049_fill_decode_whitespace_entities_in_comments.sql` backfills existing rows
- Narrow regex (`&#x20;`, `&#x09;`, `&#xA0;` and decimal equivalents) ŌĆö `&#x41;` and `&#x26;` are intentionally preserved
