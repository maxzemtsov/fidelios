# v0.0.51

> Released: 2026-05-19

The code-review loop wired end to end — reviewers self-trigger, engineers gate
on review — plus a server input-validation fix.

## The code review loop

[#85](https://github.com/fideliosai/fidelios/pull/85) — the `code_reviewer`
role shipped in v0.0.50 could not actually trigger: its bundle was purely
issue-driven and nothing created those review issues, so pull requests could
ship unreviewed.

- `code_reviewer` scaffold (`HEARTBEAT.md`, `AGENTS.md`): the reviewer now
  scans `gh pr list --state open` every heartbeat and reviews any PR that does
  not yet carry its verdict — an assigned review issue stays a fast-path
  trigger. After the verdict it comments on the PR's source issue and
  @-mentions the author.
- Default engineer Git Workflow (`onboarding-assets/default/HEARTBEAT.md` +
  repo `AGENTS.md` §11): after opening a PR the engineer creates a review
  issue assigned to the company's Code Reviewer and does not merge until the
  reviewer has approved — green CI is necessary but not sufficient.

## `X-FideliOS-Run-Id` hardening

[#86](https://github.com/fideliosai/fidelios/pull/86) — a non-UUID
`X-FideliOS-Run-Id` request header made mutating routes fail with a Postgres
`invalid input syntax for type uuid` error and return 500, because the value
flows into uuid-typed columns. The actor middleware now runs the header
through `sanitizeRunId` — a valid UUID is kept (trimmed), anything else is
dropped — so a stray header can no longer break an otherwise-valid request.

## Internal

- PRs [#85](https://github.com/fideliosai/fidelios/pull/85) and
  [#86](https://github.com/fideliosai/fidelios/pull/86) merged off `main` with

- `run-id-header-validation.test.ts` added — 6 cases covering `sanitizeRunId`
