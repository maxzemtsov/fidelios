# v0.0.40

> Released: 2026-05-06

Ten improvements across adapters, server, and UI — all shipped and validated on main.

## New: Ollama in-process adapter (Phase 1)

FideliOS now has a native **ollama-local** adapter that runs entirely in-process via the Ollama npm SDK — no external CLI subprocess, no hermes gateway needed.

- Streaming chat over `ollama.chat()` with live token output
- Auto-discovers local models via `/api/tags` and merges them with known cloud model list (Kimi K2.6, DeepSeek V4, GLM 5.1)
- Health + auth probe on every heartbeat
- 49 unit tests covering session codec, model registry, and streaming parser

To use: create an agent with adapter `ollama-local` and pick any locally-pulled model (e.g. `llama3.1`) or a supported cloud model.

## Server: auto-clear session on adapter/model swap

When you change an agent's adapter or model, FideliOS now automatically clears the in-memory runtime session before the next heartbeat. Previously, stale sessions from the old adapter/model could bleed into the new configuration.

## Adapter fixes

- **codex-local**: self-heals on resume model mismatch — detects when the thread's model no longer matches the configured model and reopens automatically instead of erroring
- **hermes-local**: stateful session codec confirmed and regression-tested — `sessionParams.sessionId` is now explicitly asserted on read and write paths

## Per-task model routing

Agents can now route different task kinds to different models without changing the global adapter config. Configure rules in agent settings; the resolver picks the best model based on task kind (`comment`, `code`, `triage`, `plan`, `research`) and falls back gracefully.

## UI

- **Model dropdown**: friendly provider labels and cloud indicator badge — Ollama Cloud models are visually distinct from local ones
- **Clear Session**: renamed from "Reset Sessions" and now shows a confirmation dialog before clearing
- **Peak Hours bypass**: one-click bypass button (1–6h) with local timezone display so you can see exactly when the bypass expires

## CI/CD

- Fixed `pr.yml` and `refresh-lockfile.yml` which were triggering on `master` instead of `main` — all PR workflows now run correctly
- Branch protection enabled on `main`: requires `policy + verify + e2e` green before any merge

## How to upgrade

```
npm install -g @fideliosai/cli@latest
fidelios --version  # 0.0.40
```
