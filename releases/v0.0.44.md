# v0.0.44

> Released: 2026-05-17

An in-app viewer for document references in agent comments, plus a dev-server shutdown fix.

## Clickable document references in comments

[#72](https://github.com/fideliosai/fidelios/pull/72) — when an agent references a workspace file in an issue comment (a filename in inline code, e.g. `PANEL_CONSENT.md`), it now renders as a clickable button instead of an inert gray span.

- Clicking opens a viewer dialog. Markdown and text files render inline (markdown formatted); the file is resolved in the issue's project workspace by direct path, or by a bounded, traversal-safe recursive search for a bare filename.
- Binary files — PDF, Excel, images, archives — are classified server-side and offered as a **Download** rather than an inline preview; the file streams to the host, where it opens with the native app.
- New `GET /companies/:companyId/issues/:issueId/files` endpoint (`?download=1` streams the raw file). The linkifier requires a real, letter-led extension, so version strings like `v1.2.3` are left alone.

## Dev-server clean shutdown

[#71](https://github.com/fideliosai/fidelios/pull/71) — `scripts/dev-runner.mjs` spawned the dev server but, on shutdown, signalled only the immediate `pnpm` wrapper — orphaning the `tsx`/`node` server beneath it. The server child is now spawned in its own process group and the whole group is terminated (SIGTERM, then a SIGKILL escalation), so `fidelios service uninstall` cleanly stops a dev service with no orphaned processes. `docs/cli/service-commands.md` documents stopping dev services and finding stray servers.
