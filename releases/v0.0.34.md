# v0.0.34

> Released: 2026-04-17
> Installer UX pass + Docker image fix. Re-verified end-to-end on fresh
> Ubuntu 24.04 + Windows Server 2022 EC2.

## Installer UX

- **All three installers now ask the user whether FideliOS should keep
  running on its own** after they close Terminal / PowerShell, restart,
  or let the laptop sleep. Language is deliberately plain — no "launchd",
  "systemd unit", or "Scheduled Task" jargon in the prompt itself.
  - If yes: runs `fidelios service install` (Mac/Linux) or registers a
    Scheduled Task (Windows). `--yes` on bash installers implies yes.
  - If no: warns that FideliOS will stop when the window closes and
    prints a link to [docs.fidelios.nl/start/keep-running](https://docs.fidelios.nl/start/keep-running).
- New CLI flags on bash installers: `--service`, `--no-service`.
- `install.sh` also skips auto-service in pure pipe mode (`curl | bash`
  without `--yes`) to avoid silently registering launchd for users who
  didn't consent.

## Docs

- New [docs/start/keep-running.md](https://docs.fidelios.nl/start/keep-running)
  covers every OS path:
  - **macOS**: `fidelios service install` (launchd), Amphetamine 5.3.2
    trigger setup (free App Store), `caffeinate -i fidelios run`
    fallback.
  - **Linux**: `fidelios service install` (systemd user unit) +
    `loginctl enable-linger` for headless servers.
  - **Windows**: Task Scheduler step-by-step, nssm for real services,
    PowerToys Awake as sleep-prevention.
- Added to docs.json Introduction nav between `first-time-mac` and
  `troubleshooting`.

## Fixes

- **Docker image was broken since hermes-local adapter landed** —
  `ghcr.io/fideliosai/fidelios:latest` exited immediately with
  `Cannot find package '@fideliosai/adapter-utils'` imported from
  `hermes-local`. Root cause: Dockerfile's `COPY packages/adapters/*`
  list was missing `hermes-local/package.json`, so pnpm install never
  linked its workspace deps. Fixed.
