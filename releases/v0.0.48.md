# v0.0.48

> Released: 2026-05-17

Hiring a new agent now produces a complete, role-specific instruction package — and the human reviewer sees the whole package in the approval before deciding.

## Fix: agent hires shipped a one-file instruction bundle

[#78](https://github.com/fideliosai/fidelios/pull/78) — a managed instruction bundle is four files (`AGENTS.md`, `SOUL.md`, `HEARTBEAT.md`, `TOOLS.md`), but only the `ceo` role scaffolded all four. Every other hire received `AGENTS.md` alone, and a supplied `promptTemplate` collapsed straight into that one file, suppressing the rest. The reviewer also never saw the instruction content — the approval carried only config paths.

- **Scaffold parity** — every role now scaffolds all four bundle files, not just `ceo`. New `onboarding-assets/default/{SOUL,HEARTBEAT,TOOLS}.md` starter templates back this.
- **`instructionFiles` on the hire request** — `POST …/agent-hires` accepts an `instructionFiles` map so the hiring agent (the CEO) authors each file. Files provided override the scaffold; files omitted keep it. A legacy `promptTemplate` still becomes `AGENTS.md` but no longer suppresses the other three.
- **Approval visibility** — a new `GET /agents/:id/instructions-bundle/files` endpoint; the hire approval now renders the full materialized instruction package (collapsible, entry file open) alongside capabilities and chosen skills.

## Skill: the CEO authors all four files and triages skills

The `fidelios-create-agent` skill is rewritten: the hiring agent must author all four instruction files for the role and triage the company skill library (`desiredSkills`) as a required step, rather than relying on a single vague `promptTemplate`.
