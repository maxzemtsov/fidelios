# v0.0.26

> Released: 2026-04-06

## Fixes

- **Non-interactive install script** — `curl | bash` installs no longer block on interactive `[y/N]` prompts. TTY detection gates all user prompts; piped installs auto-accept defaults.
- **npm EACCES permission fix** — Install script uses `~/.npm-global` prefix when `/usr/local` is not writable, preventing `EACCES` errors on fresh macOS setups.
- **Post-install PATH persistence** — PATH updates written to `~/.zprofile` / `~/.bash_profile` so `fidelios` command is available after `source ~/.zprofile`.
- **Bash shell warning** — Users whose default shell is bash now see an informational note recommending `chsh -s /bin/zsh`.
- **Onboarding CLI install button** — When the adapter environment check fails because a CLI tool is missing, the onboarding wizard now shows an Install button (matching the agent config form behavior).
- **Telegram plugin discovery** — `@fideliosai/plugin-telegram-gateway` is now registered in the bundled plugin registry, making it discoverable in the Plugin Manager UI.
- **Gemini adapter in edit view** — `gemini_local` adapter now appears in the adapter type dropdown when editing/reconfiguring an existing agent (previously only shown during creation).
- **Gemini hello probe timeout** — Default probe timeout increased from 10s to 45s to accommodate OAuth-based auth flows.
- **Peak hours banner visibility** — The peak hours banner now shows at all times when peak hours are configured, with an amber "active" state during the window and a muted blue "scheduled" state outside it.
- **Default peak hours on new companies** — New companies are created with default peak hours configuration so the banner is visible from day one.
