# v0.0.38

> Released: 2026-04-21
> Plugin activation fix for Node.js 24+ — the bundled example plugins no longer expose raw TypeScript to the runtime loader.

## What's fixed

On Node.js 24 and newer, clicking **Install Example → Telegram Gateway**
(or any other bundled plugin) failed worker initialisation with:

```
Error [ERR_UNSUPPORTED_NODE_MODULES_TYPE_STRIPPING]:
  Stripping types is currently unsupported for files under node_modules,
  for "file:///.../fidelios/node_modules/@fideliosai/server/
       packages/plugins/examples/telegram-gateway/
       node_modules/@fideliosai/plugin-sdk/
       node_modules/@fideliosai/shared/src/index.ts"
```

Node 22 silently stripped types from `.ts` files under `node_modules`;
Node 24 refuses and throws. The bundled example plugins inside the
`@fideliosai/server` tarball carried a pnpm-copy of each plugin's
`node_modules/`, and those nested `@fideliosai/*/package.json` files
still advertised the workspace-dev `exports` pointing at
`./src/index.ts` — because we had copied them in place instead of
routing them through `npm publish` (which would have rewritten
`exports` via `publishConfig`).

## Fix

A new release-time step post-processes the bundled `node_modules/` to
match what npm would have produced at publish:

- `scripts/sanitize-bundled-plugin-deps.mjs` walks every
  `node_modules/@fideliosai/<name>/package.json` inside the bundled
  plugin examples, hoists the package's own `publishConfig.exports`,
  `main`, `types`, `bin`, etc. into the top-level, removes the now-stale
  `publishConfig` block, and drops the unused `src/` directory.
- `scripts/release.sh` calls the sanitiser right after copying the
  examples, so every future `fidelios` release has a
  Node-24-compatible bundle.

Affects all five shipped example plugins:

- `@fideliosai/plugin-telegram-gateway`
- `@fideliosai/plugin-hello-world-example`
- `@fideliosai/plugin-file-browser-example`
- `@fideliosai/plugin-kitchen-sink-example`
- `@fideliosai/plugin-authoring-smoke-example`

## How to pick up the fix

```
npm install -g fidelios@latest
```

Then open FideliOS → **Settings → Plugins → Install Example** on the
plugin of your choice. Worker initialisation completes cleanly on
Node 18, 20, 22, and 24.

## Who was affected

Anyone installing FideliOS on a machine with **Node.js 24 or newer**.
Node 20/22 users were unaffected because those runtimes silently
stripped types from `.ts` files under `node_modules`.
