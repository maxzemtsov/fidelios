> Released: 2026-05-18

A dedicated `code_reviewer` agent role — independent pull-request review, on a different model from the authoring agents.

## The `code_reviewer` role

[#83](https://github.com/fideliosai/fidelios/pull/83) — Pillar 3 of safe parallel multi-agent development: a new agent role for an independent review gate on every pull request.

- `code_reviewer` added to `AGENT_ROLES` and `AGENT_ROLE_LABELS`.
- A role-specific four-file instruction bundle (`onboarding-assets/code_reviewer/`) — the reviewer's playbook: triggered by a review issue, inspects the PR with `gh`, works a review checklist (correctness, tests, Git Workflow compliance, safety, scope), records the verdict via `gh pr review`, and never authors code. Wired into `DEFAULT_AGENT_BUNDLE_FILES` so the role gets its own scaffold.

A reviewer agent is intended to run on the `codex_local` adapter — a genuinely different model from the Claude-adapter agents that write the code, so it catches a different class of issue than self-review would.
