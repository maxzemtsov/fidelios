# v0.0.12

> Released: 2026-04-01

## Highlights

- **Orphaned process recovery** — Agent runs no longer hang forever after server restarts. Graceful shutdown kills all child processes before exit. Startup reconciliation marks stale runs as failed and queues automatic retry. Cancel button works even when the in-memory process handle is lost (PID-based fallback).
- **Level 1 context caching** — Claude adapter reuses a persistent per-agent skills directory across heartbeat runs instead of rebuilding a tmpdir each time. OpenCode adapter caches instructions files in memory with mtime-based invalidation.
- **Routine Telegram notifications** — Routine execution results are now routed to a dedicated "Routines" Telegram topic instead of agent board topics.
- **Project-scoped routines** — When a routine has a project assigned, the project name is injected into the issue description so agents focus only on that project.

## Fixes

- **Agent inbox for routines** — `inbox-lite` now includes `routine_execution` issues so agents can discover and process their assigned routine tasks.
- **Read-before-Write directive** — OpenCode adapter injects a rule requiring agents to Read files before Edit/Write, preventing "must read first" errors with models that don't follow this convention.
- **Skills cleanup** — Removed duplicate and stale skill entries. Shared skills (op-secrets, aws-cli, etc.) registered with direct paths. Old paperclip references cleaned up.
- **Docker CI** — Docker image builds now only trigger on minor/major releases (v0.X.0), not on every push to main.

## Advisory

**Codex Local Adapter**: The `codex_local` adapter has significantly higher token consumption (~936K input tokens/run) compared to `claude_local` (~37/run) or `opencode_local` (~211K/run). This is an architectural limitation — Codex CLI does not support session persistence. We recommend using `claude_local` or `opencode_local` with local Ollama models until the Level 2 context caching middleware ships. See [ROADMAP.md](../ROADMAP.md) for the full caching plan.
