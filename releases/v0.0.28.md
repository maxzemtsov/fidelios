# v0.0.28

> Released: 2026-04-17

## Fixes

- **macOS background service (`fidelios service install`)** — the launchd plist
  now sets `RunAtLoad=true` and `KeepAlive=true` (previously `KeepAlive` only
  restarted on non-zero exit, so clean exits left the service dead), runs
  `launchctl kickstart -k` immediately after `load`, and seeds the service
  `PATH` with `~/.claude/local/bin`, `~/.codex/bin`, `~/.cargo/bin`,
  `~/.npm-global/bin`, and `/opt/homebrew/bin`. Fixes *"Command not found in
  PATH: claude"* in heartbeat execution when FideliOS runs under launchd.
- **Linux service** — matching systemd unit hardened with `Restart=always` and
  the same `PATH` seeding.
- **Windows installer** — `install.ps1` now installs Node.js via `winget` (or
  nvm-windows as fallback) and runs `npm install -g fidelios@latest`, matching
  the docs and the macOS flow. Previous script pulled a Docker image that was
  never actually published for patch releases.
- **Docker CI** — workflow triggers on every `v*` tag and main pushes; `:latest`
  tracks the most recent non-prerelease tag, `:nightly` tracks main, and each
  release produces `:X.Y.Z` / `:X.Y` / `:sha-xxxxx` tags. Earlier the workflow
  only ran for `v*.0.0` tags and `:latest` was never produced.
- **`scripts/kill-dev.sh`** — no longer skips PostgreSQL, extends process
  matching to embedded-postgres workers and plugin workers, sweeps ports
  3100-3110 / 5173 / 54331, and cleans stale `postmaster.pid` / `.lock` files
  so the next `fidelios run` starts cleanly.
- **Install script non-interactive path** — `curl | bash` now runs
  `fidelios onboard --yes` instead of leaving the user without a config.
- **Port conflict UX** — in non-interactive shells (CI, launchd, `--no-tty`
  pipes) `fidelios run` now fails fast with the conflicting PID instead of
  silently bumping to 3101+. Set `FIDELIOS_ALLOW_PORT_BUMP=true` to restore
  the legacy bump behaviour.
- **DB migrations on upgrade** — embedded-postgres migrations now auto-apply
  on every startup, not just first-run. Previous behaviour left the schema
  stale after `npm install -g fidelios@latest`.

## New

- **`fidelios stop`** — first-class CLI command that walks the fidelios process
  tree (server, embedded-postgres and its workers, plugin workers), sends
  SIGTERM then SIGKILL to stragglers, and removes stale lock files. Flags:
  `--service` also unloads launchd/systemd, `--dry-run` previews without
  killing. Replaces the ad-hoc `fidelios-killall` shell alias.

## Dev experience

- **`autoRestartDevServerWhenIdle` now works in `pnpm dev` (watch mode)** —
  previously the toggle only applied to `pnpm dev:once` because watch mode
  delegated to tsx's built-in watcher. The dev-runner now does its own
  file watching in both modes so the toggle is honoured everywhere. Decision
  matrix: `dev:once + off` = manual, `dev:once + on` = wait-for-idle,
  `watch + off` = immediate restart, `watch + on` = wait-for-idle.
- **DevRestartBanner copy** updated to reflect the multi-mode flow.

## Docs

- [docs/start/install/windows.md](https://docs.fidelios.nl/start/install/windows)
  now matches the actual installer behaviour (Node+CLI, not Docker).
- [docs/cli/service-commands.md](https://docs.fidelios.nl/cli/service-commands)
  describes the current plist behaviour and documents `fidelios stop`.
