# v0.0.36

> Released: 2026-04-21
> Windows fix: Codex / Claude / any .cmd adapter now works when the npm global path contains a space (e.g. `C:\Users\Sergey F\...`).

## What's fixed

Spawning any `.cmd` or `.bat` adapter CLI on Windows failed when the
path contained whitespace — very common because the default npm global
prefix is `C:\Users\<full name>\AppData\Roaming\npm\`, and "First Last"
or display names with a space land right inside it.

Symptoms users were seeing in the **Test Environment** panel:

```
ERROR · Codex hello probe failed.
        ("C:\Users\Sergey F\AppData\Roaming\npm\codex.CMD\"  ♦♦♦♦…)
```

The garbled quotes and diamond glyphs were the result of two stacked
bugs in `@fideliosai/adapter-utils`:

1. **Node re-escaped the command line.** We built the correct quoted
   form `"<exe>" args` but spawned `cmd.exe /d /s /c <cmdline>`
   without `windowsVerbatimArguments: true`. Node then re-applied
   Windows escaping and turned internal `"` into `\"`, giving CMD an
   input it couldn't parse (the `codex.CMD\"` visible above).
2. **`cmd /s /c` stripped our only quotes.** Per `cmd /?`, `/s /c`
   removes the first and last quote character. Our only pair was
   around the exe path, so stripping collapsed
   `"C:\Users\Sergey F\…"` down to a bare `C:\Users\Sergey F\…`, and
   CMD then tried to execute `C:\Users\Sergey` with `F\…` as the
   first argument.

## Fix

Use the canonical `cmd /d /s /c ""<exe>" <args>"` form with an **outer**
pair of quotes so `/s /c` strips those and leaves the inner quotes
around the whitespace-containing exe path intact, combined with
`windowsVerbatimArguments: true` so Node doesn't re-escape.

Applies to every local CLI adapter that routes through
`runChildProcess` — Codex, Claude Code, Gemini, Codex Coder, and any
`.cmd`/`.bat` custom adapter.

## How to pick up the fix

On Windows:

```powershell
iwr -useb https://fidelios.nl/install.ps1 | iex
```

or just upgrade the CLI directly:

```
npm install -g fidelios@latest
```

Then open FideliOS, go to your Codex (or other CLI-backed) agent →
**Test Environment** → the hello probe should now pass.

## Also in this release

- New beginner-friendly
  [Codex Local adapter docs](https://docs.fidelios.nl/adapters/codex-local/)
  with per-OS install instructions, ChatGPT vs API-key auth walkthrough,
  and a troubleshooting section covering the top failure modes.
- Landing-page install section split Windows npm commands into two
  clean lines (no more trailing `;` that CMD doesn't understand).
- `install.ps1` now sets `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
  before invoking npm so users on a fresh Windows with the default
  Restricted execution policy don't hit `PSSecurityException` from
  `npm.ps1`.
